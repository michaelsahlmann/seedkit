# Principios tRPC

> Seguridad de tipos end-to-end para monorepos TypeScript.

## Cuándo Usar

```
✅ Fit perfecto:
├── TypeScript en ambos extremos
├── Estructura de monorepo
├── Herramientas internas
├── Desarrollo rápido
└── Seguridad de tipos crítica

❌ Mal fit:
├── Clientes no TypeScript
├── API pública
├── Necesitas convenciones REST
└── Backends en múltiples lenguajes
```

## Beneficios Clave

```
Por qué tRPC:
├── Cero mantenimiento de schema
├── Inferencia de tipos end-to-end
├── Autocomplete IDE a través del stack
├── Cambios de API reflejados instantáneamente
└── Sin paso de generación de código
```

## Patrones de Integración

```
Setups comunes:
├── Next.js + tRPC (más común)
├── Monorepo con tipos compartidos
├── Remix + tRPC
└── Cualquier frontend TS + backend
```
